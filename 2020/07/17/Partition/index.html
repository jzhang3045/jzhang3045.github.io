Today July’s LeetCode challenge gives me a chance to revisit a problem, which is interesting.

LC347 Top K Frequent Elements

Given a non-empty array of integers, return the k most frequent elements.

Example 1:

Input: nums = [1,1,1,2,2,3], k = 2
Output: [1,2]

Example 2:

Input: nums = [1], k = 1
Output: [1]

My solutions:

1. using heap

Every time I encounter the key word “most” or “least”, I know that this problem could be solved with heap. At here, we want the most frequent elements, so it’s quite straightforward to use max heap. The implementation is also very simple:

class Solution {
    public int[] topKFrequent(int[] nums, int k) {
        //buiild a max heap which is ordering with elements' frequency
        var queue = new PriorityQueue<Map.Entry<Integer, Integer>>((el1, el2)->
                                                                 el2.getValue() - el1.getValue());
        //use hashmap to count each elements' frequency
        var map = new HashMap<Integer, Integer>();
        int[] res = new int[k];
        for (int num : nums) {
            if (map.containsKey(num)) map.put(num, map.get(num) + 1);
            else map.put(num, 1);
        }
        for (Map.Entry<Integer, Integer> element : map.entrySet()) {
            queue.add(element);
        }
        //take out the first k elements and put them into our result
        for (; k > 0; k--) {
            res[k - 1] = queue.poll().getKey();
        }
        return res;
    }
}

Time complexity will be $O(nlog(n))$.

2. With quickselect (Lomuto partition)

This solution is based on the article of Leetcode’s solution. with quick select technique, we don’t need to gain the $k$ most frequent elements in order, so we are in good chance to have less time time complexity $O(n)$, although sometimes we may encounter the worst case which is $O(n^2)$.

class Solution {
    //get one unique array
    int[] unique;
    //used to contain the number for every unique number;
    Map<Integer, Integer> count;
    public int[] topKFrequent(int[] nums, int k) {
        count = new HashMap();
        for (int num : nums) count.put(num, count.getOrDefault(num, 0) + 1);
        //build a unique array
        int n = count.size();
        unique = new int[n];
        int i = 0;
        for (int num: count.keySet()) {
            unique[i] = num;
            i++;
        }
        quickselect(0, n - 1, n - k);
        return Arrays.copyOfRange(unique, n - k, n);
    }

    //swap function preparation
    public void swap(int a, int b) {
        int tmp = unique[a];
        unique[a] = unique[b];
        unique[b] = tmp;
    }
    //lomuto's partition algorithm
    //return pivot's position
    public int partition(int lo, int hi) {
        int pivot = unique[hi];
        int i = lo;
        for (int j = lo; j <= hi; j++) {
            if (count.get(unique[j]) < count.get(pivot)) {
                swap(i, j);
                i++;
            }
        }
        swap(i, hi);
        return i;
    }    
    public void quickselect(int left, int right, int k_smallest) {
        if (left == right) return;
        int pivot_index = partition(left, right);
        if (k_smallest == pivot_index) return;
        else if (k_smallest < pivot_index) quickselect(left, pivot_index - 1, k_smallest);
        else quickselect(pivot_index + 1, right, k_smallest);
    }
}
