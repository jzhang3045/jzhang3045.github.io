Description of this problem:

72. Edit Distance

Given two words word1 and word2, find the minimum number of operations required to convert word1 to word2.

You have the following 3 operations permitted on a word:

  1. Insert a character
  2. Delete a character
  3. Replace a character

Example 1:

Input: word1 = "horse", word2 = "ros"
Output: 3
Explanation: 
horse -> rorse (replace 'h' with 'r')
rorse -> rose (remove 'r')
rose -> ros (remove 'e')

Example 2:

Input: word1 = "intention", word2 = "execution"
Output: 5
Explanation: 
intention -> inention (remove 't')
inention -> enention (replace 'i' with 'e')
enention -> exention (replace 'n' with 'x')
exention -> exection (replace 'n' with 'c')
exection -> execution (insert 'u')

My understand:

First try:

At beginning, I notice that LCS(longest common subsequence) could be used here, for example in example 1:

word1 = "horse", word2 = "ros";
their LCS = "os";
thus except these common subsequence, word1 should replace, delete or replace in order to become word2;

And here is my implementation:

class Solution {
    public int minDistance2(String word1, String word2) {
        int l1 = word1.length();
        int l2 = word2.length();
        int[][] table = new int[l2 + 1][l1 + 1];
        int max = 0;
        for (int i = 0; i < l2; i++) {
            for (int j = 0; j < l1; j++) {
                if (word1.charAt(j) == word2.charAt(i)) table[i + 1][j + 1] = table[i][j] + 1;
                else table[i + 1][j + 1] = Math.max(table[i][j + 1], table[i + 1][j]);
                max = Math.max(max, table[i + 1][j + 1]);
            }
        }
        return Math.max(l1,l2) - max;
    }
}

Input:"intention" "execution"

The reason is that, in this way I try to reuse characters of common subsequence and only modify others. However, not all of the subsequence could be reused. In this case, the LCS is "etion", but "e" in the wrong position, and cannot be excepted from additional operation.

The right way

After reading solution I realize this is a classic problem, and there is a famous algorithm (Levenshtein distance) behind it:
$$
\operatorname{lev}{a, b}(i, j)=\left{\begin{array}{ll}\max (i, j) & \text { if } \min (i, j)=0 \ \min \left{\begin{array}{ll}\operatorname{lev}{a, b}(i-1, j)+1 & \ \operatorname{lev}{a, b}(i, j-1)+1 & \text { otherwise } \ \operatorname{lev}{a, b}(i-1, j-1)+1_{\left(a_{i} \neq b_{j}\right)}\end{array}\right.\end{array}\right.
$$
Based on this, I write another:

public int minDistance(String word1, String word2) {
    int l1 = word1.length();
    int l2 = word2.length();
    if (l1 == 0 || l2 == 0) return l1 + l2;
    int[][] table = new int[l2 + 1][l1 + 1];
    for (int i = 0; i < l2 + 1; i++) table[i][0] = i;
    for (int i = 0; i < l1 + 1; i++) table[0][i] = i;
    for (int i = 1; i < l2 + 1; i++) {
        for (int j = 1; j < l1 + 1; j++) {
            int top = table[i - 1][j] + 1;
            int left = table[i][j - 1] + 1;
            int topleft = (word1.charAt(j - 1) == word2.charAt(i - 1))? table[i - 1][j - 1] : table[i - 1][j - 1] + 1;
            table[i][j] = Math.min(top, Math.min(left, topleft));
        }
    }
    return table[l2][l1];
}

Interesting observations to share

Why in (1) we take the minimum between these three entries:

Here is explanation:

word1 = "inte";
word2 = "ex";
Suppose at some point we try to get transform from "inte" -> "ex":
1."inte" -> "e";            //this case corresponds to adding another "x"
2."int"  -> "ex";           //this case corresponds to deletion
3."int"  -> "e";            //this case corresponds to substitution, if word1[i] == word2[j], no substitution needed
